Web Servers

The Index file concerning 'Websites' is the main page or default page that any given server will display for the user if their request does not specify a page in their request. The 'index.html' file typically serves as the default page on web servers, such as Oregon State University's Apache server, and is returned to the user when no specific page is requested. This file, usually located at the root directory, becomes the web application's homepage. However, some servers, like those on Microsoft's .NET platform, might use 'default.html' instead.

When the 'index.html' file is opened locally, the file path is displayed in the URL bar and 'Sources' tab of the Web Dev/Inspector tools, with no HTTP(S) headers, since it's not fetched via the web. Even though this 'index.html' file has been opened locally, one may still see the Status 200 when viewing the Network tab. However, when the same file is accessed through a web server, the 'Sources' tab shows the Server Domain instead of the local file path. On the 'Network' tab, one would observe two additional objects not seen locally - an 'executor.js' script and a 'favicon.ico' file. Also, instead of the 'index.html' file seen locally, the document file named 'a1-username' is displayed. This is accompanied by HTTP status codes like 404 for not found resources, reflecting the state of the requested web resources.

The favicon.ico file returned a status of 200, meaning that the server has found the favicon.ico file and returned it to the user. Since it may be deduced that the file was not placed by the creator of the index.html, the hosting server automatically added the favicon.ico file. While the main.js and main.css files returned a status of 404 because the files are not present in the directory. We expect the latter two files to have the ‘Not Found’ status, as they were not placed in the directory initially.

The scheme will identify the protocol that will need to be used to access this web page ('https'). The subdomain is the subset of the original domain. This is used to better navigate the different sections of the website ('web.engr'). The host domain is the main location in which the website is hosted ('oregonstate.edu'). The resources are typically used to request specific pages or contents to be displayed back to the user. In this case, the '/~partidju' indicates a particular user’s directory, while the 'a1-partidju' indicates the specific folder within the referenced directory. When all the components are put together, the complete URL is https://web.engr.oregonstate.edu/~partidju/a1-partidju/.

Frontend Design

The concept of Frontend Design focuses on how user-friendly, intuitive and attractiveness interface a website may be. High-quality content, interactive features, aesthetically pleasing design, and other aspects that grab readers' attention and keep them on the website can accomplish this.

Effectiveness
Measuring the success of a user in achieving their task through the website. An effective website supplies users with the information they need in a clear and concise manner, facilitating the completion of their intended tasks.
Efficiency
This measures the ease and speed with which a user can accomplish a task on a website. The quicker a user can navigate a website and find the required information, the better the user experience.
Navigation Ease
This relates to the simplicity of a user's familiarity with the website. A well-designed website will be intuitive and easy to understand, even for beginners. This can be achieved through straightforward navigation, consistent design elements, and effective web design principles.
Error Tolerance
This term represents a website's ability to either anticipate user errors or respond to them effectively when they occur. A robust website minimizes the occurrence of errors, provides appropriate error messages when they do happen, and facilitates easy recovery.
Engagement
This term focuses on the website's user-friendliness or attractiveness. This can be accomplished through high-quality content, interactive elements, visually pleasing design, and other features that captivate users and keep them engaged on the website.

HTML employs anchors, denoted by <a>, to generate links, establishing connections between different content types. Three primary usages for anchors include linking to external websites, referencing page sections internally, and navigating between pages within the same website.

Anchor Link Usage Examples:

External Example: These point to other websites, with the `href` attribute specifying the target URL. For example: `Visit Example.com`.

Internal Example: These refer to sections within the same page, accomplished by matching the `href` attribute with a unique id on the page. For example: `Go to Section 1`, paired with an element labeled id="section1".

Inter-Page Example: These allow navigation between pages on the same site, using relative paths. For instance, to link to a "contact.html" page in the same directory: `Contact Us`.

Page Layout Tags:

Header
The <header> element represents a container for introductory content or a set of navigational links. It may contain some heading elements but also other elements like a logo, wrapped section's heading, a search form, and so on. Search engine robots can identify the primary subject of each page by reading the content within the header tags.
Nav
The <nav> element is intended for major block of navigation links. Not all groups of links on a page need to be in a <nav> element only sections that consist of major navigation blocks are appropriate for the <nav> element. For search engines, the nav tag can help identify which links are primary versus secondary.
Main
The <main> element is used to encapsulate the primary, or main, content of the webpage. It is unique to the document and excludes content that is repeated across a set of documents such as site navigation links, header or footer information. For search engines, this tag helps identify the core topic of a page.
Section
The <section> element represents a standalone section which doesn't have a more specific semantic element to represent it. It usually has a heading. For search engines, this helps in understanding the structure of the page and the importance of the content within the section.
Article
The <article> element represents a complete, or self-contained, composition in a document, page, application, or site and that is, in principle, independently distributable or reusable. For search engines, this helps in distinguishing the main content from the rest of the page content.
Footer
The <footer> tag defines a footer for a document or section. It should contain information about the author, copyright information, etc. Search engines can use this to identify supplementary information about the page.

Optimizing Images

Optimized images meet these specifications:

Descriptive file name
To improve search engine optimization (SEO), file names should include who, what, when, and where as much as possible. Use descriptive but concise file names. For example, a picture of a German Shepard puppy playing with a ball could have a file name like this: german-shepard-puppy-ball.jpg. This helps search engine bots categorize the image for users searching for related content.
Small file size
Reduce file sizes as much as possible to improve load time. Serve high-resolution images only to high-resolution devices, typically using the srcset="" and sizes="" attributes. Consider the differences between lossy and lossless compression. Lossy compression uses approximations to discard some image data, which can result in pixelation, while lossless compression does not degrade visual quality.
Exact dimensions
Crop and reduce images to fit the dimensions of the space in your web page. Serving an image much larger than its display size will increase load time.
Correct file format
Use appropriate file formats for different types of images. Photos are usually in .JPG format. Line-art images, like logos and icons, are usually .GIF and sometimes 8-bit .PNG. Graphics requiring true transparency need 24-bit .PNG.
Reduced resolution
Monitors render between 72 or 300+ pixels per inch (ppi); the older default is 72ppi. As higher resolutions become more common, providing multiple image sizes for each is becoming standard practice.
Color Mode
Use RGB for .PNG, .JPG, .SVG, and .WebP files, and Indexed for .GIF files.
Appropriate File Formats for photos and for line art:
Photos (.JGP)
.JGP is best for photos due to its efficiency in compressing complex images like photographs without noticeable loss in quality for most viewers.
.JGP supports millions of colors and is widely supported by web browsers and image editing software.
Line Art (.GIF and .PNG)
.GIF format is suitable for line-art images like logos and icons.
.GIF renders sharp lines and limited color palettes efficiently, without unwanted artifacts.
.GIF supports transparency and is widely supported by web browsers.
.PNG Is also suitable for line art images as it can handle more colors than GIF whilst preserving transparency without loss of quality.

Favicons

A favicon, short for "favorite icon", is a small icon associated with a website or web page. It is typically displayed in the address bar of a browser, or next to the site name in a user's list of bookmarks.

Using a favicon provides several advantages:

  • Branding: A favicon helps users to easily identify your website if they have multiple tabs open. It contributes to the overall branding of your site.
  • Usability: It helps users locate a webpage they want to return to, by visually identifying the site in bookmarks, history, and on open tabs.
